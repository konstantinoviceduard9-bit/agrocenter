# Агенты и автопилот для репозитория «Нерал — дашборд»

Этот файл читают **Cursor Agent / Cloud Agent** и люди. Цель: любой прогон «улучши всё» был предсказуемым: что проверять, что не ломать, какие команды гонять.

## Команды проверки (обязательно перед merge)

Из каталога `web/`:

```bash
npm ci
npm run check
```

`check` = ESLint + production build (см. `web/package.json`). Локально достаточно `npm run check` после `npm ci`.

## Ограничения (не нарушать)

- **GitHub Pages**: приложение собирается с `VITE_BASE_PATH=/<repo>/` в CI (см. `.github/workflows/deploy-github-pages.yml`). Не ломать `BrowserRouter basename`, относительные пути к `public/data/`, SPA `404.html`.
- **Данные**: снимок `web/public/data/dashboard.snapshot.json` опционален; при отсутствии — моки из кода. Не удалять fallback и не требовать файл в рантайме.
- **Объём правок**: править только то, что относится к задаче; не делать массовый рефакторинг без явного запроса.

## Роль A — «QA / сборка»

**Задача:** зелёный пайплайн, типы, линт.

**Чеклист:**

1. `cd web && npm run check`
2. Если падает ESLint — исправить ошибки (не отключать правила без причины).
3. Убедиться, что `npm run build` проходит с тем же `VITE_BASE_PATH`, что в CI (при сомнениях — как в workflow).

**Готово, когда:** `npm run check` exit 0.

---

## Роль B — «UI / UX»

**Задача:** единообразие интерфейса, мобилка, a11y поверхности.

**Чеклист:**

1. Все страницы с `PageShell`: крошки и `DataSourceStrip` согласованы; нет дублирующих «источник данных» в длинных подзаголовках без нужды.
2. **Мобилка (`< lg`)**: сайдбар только в дроуере; контент `min-w-0`, нет горизонтального переполнения на типовых экранах.
3. Карточки: классы `surface-card` / `surface-card--lift` из `web/src/index.css` вместо разрозненных `border rounded-2xl`.
4. Фокус: интерактивные элементы видны при клавиатурной навигации (`:focus-visible` в базовых стилях).
5. Таблицы: `DataTable` — залипание шапки, читаемые отступы на узкой ширине.

**Готово, когда:** визуальный проход ключевых маршрутов (`/`, `/finance/*`, `/operations/alerts`, `/company/:id`, `/admin/settings`) без явных артефактов.

## Роль V — «Визуал / полировка»

**Задача:** эстетика и согласованность с презентациями (`docs/presentation-neral-exec-ai-dashboard.html`, `docs/presentation-neral-product-sales.html`): палитра, типографика, графики, тени, движение.

**Чеклист:**

1. **Палитра:** акцент emerald (`#047857` / `#059669`), нейтральный slate для текста и сеток — как в deck; не вводить новые «случайные» цвета без причины.
2. **Графики (Recharts):** цвета и сетка из `web/src/lib/chartPalette.ts`; при новом графике — расширять этот файл, не хардкодить hex в компонентах.
3. **Название продукта:** только через `web/src/components/BrandWordmark.tsx` (локап «Нерал · дашборд» + опциональный tagline); не дублировать строкой в `h1` без нужды. Шрифт **Inter** подключён в `web/index.html` и первым в стеке в `index.css` — как в `docs/presentation-neral-exec-ai-dashboard.html`.
4. **Карточки и кнопки:** `surface-card`, `btn-ghost` из `web/src/index.css`; вторичные действия (CSV, «Выйти») выглядят одинаково.
5. **Движение:** `motion-reduce:` для выезжающего меню и оверлея; `prefers-reduced-motion` — отключать декоративные `transition` у `surface-card--lift` (см. конец `index.css`).
6. **Микровзаимодействия:** hover по строкам списков/таблицам, лёгкий `active:scale` на pill-ссылках только с `motion-reduce` откатом.
7. **Иерархия:** подзаголовки секций — единый паттерн `text-xs font-semibold uppercase tracking-wide text-slate-500`.

**Готово, когда:** сводка и финансовые экраны выглядят цельно на desktop и mobile, без «ломаных» графиков и лишней анимации при reduced motion.

## Роль C — «Продукт и копирайт»

**Задача:** понятные формулировки, без противоречий с доками.

**Чеклист:**

1. Подписи «мок», «снимок», «API позже» — последовательные по приложению.
2. Сверка ключевых сообщений с `docs/presentation-neral-exec-ai-dashboard.html` (термины, цифры-заглушки не выдавать за факт).
3. Ошибки загрузки снимка — коротко и по делу (см. `DataSourceStrip`).

**Готово, когда:** новый читатель понимает, откуда цифры и что ещё не подключено.

## Роль D — «Интеграция / деплой»

**Задача:** деплой и окружение.

**Чеклист:**

1. После изменений в `web/` — push в `main` поднимает GitHub Actions deploy (если репозиторий **public** для бесплатного Pages).
2. Проверить, что в артефакте есть `404.html` = копия `index.html` (шаг в workflow уже есть).
3. Не коммитить секреты; переменные окружения — только через CI secrets при появлении API.

## Как запускать «всё подкорректировать» одним заходом

1. Запустить агента с указанием: *«Выполни роли A→V→B→C по `AGENTS.md`, правки только в `web/`, в конце `npm run check`»*. Только визуал: *«A→V, затем `npm run check`»*.
2. Либо открыть PR и дождаться workflow **Quality** (линт + build).

Порядок ролей: **A** всегда первой; **V** (визуал) — после A и до merge; **B** и **C** можно совмещать с V одним проходом, если не конфликтуют.

- Название продукта: `web/src/components/BrandWordmark.tsx` + Inter в `index.html` / `index.css`.
