# Neral — план разработки

> Копия плана из репозитория. Оригинал: `docs/plan-dalneyshaya-razrabotka.md` — при правках можно синхронизировать вручную или держать одну «истину» в Obsidian и переносить в git раз в неделю.

## 0. Что уже есть

- **Фронтенд:** Vite + React + TS, Tailwind, React Router, Recharts; сводка по группе, список компаний, карточка юрлица.
- **Данные:** файл `web/src/data/companies.ts` — мок под UI (не источник правды).
- **Материалы:** в `docs/neral-vault/` — структура группы, идеи метрик, концепт TG-дашборда.

## 1. Зафиксировать цель и границы (1–3 дня, без кода)

Согласовать с заказчиком / гендиректором:

| Вопрос | Зачем |
|--------|--------|
| Какие **юрлица** в первой волне (MVP) | Объём интеграций |
| Какие **5–10 метрик** обязательны на главном экране | Приоритет разработки |
| **Источник правды** (1С редакция, СБИС, Excel, вручную) | Архитектура загрузки |
| **Доступ:** только внутри сети / VPN / публичный с логином | Безопасность и хостинг |
| Нужен ли **параллельно** Telegram или только веб на старте | Разделение работ |

Результат: короткое ТЗ в 1–2 страницы или список в задаче.

## 2. Бэкенд и контракт API (1–2 недели)

**Цель:** фронт читает данные не из мока, а из вашего API.

1. Стек бэкенда (**Node Fastify** или **Python FastAPI**) — сервис `api/`.
2. Контракт: `GET /companies`, `GET /companies/:id`, `GET /group/summary`, позже метрики с периодом.
3. Первая выгрузка: CSV/Excel, 1С OData/файл, или ручная форма.
4. Фронт: `VITE_API_URL`, `fetch` вместо мока.

**Готово, когда:** сводка из API; мок опционально для dev.

## 3. Углубление дашборда (2–4 недели)

Фильтры по периоду; виджеты дебиторка/кредиторка/ДС при наличии данных; экспорт; роли и вход.

## 4. Надёжность

Логи, бэкапы, CI (`npm run build`), Docker для web+api.

## 5. Уведомления

Пороги алертов; email или Telegram (allowlist); планировщик.

## 6. Деплой

OVH1: api + статика, HTTPS, секреты в env.

## 7. ИИ-слой

После стабильных цифр: RAG по документам, безопасные отчёты по деньгам, голос позже.

## 8. Очередь

```text
ТЗ → API + выгрузка → дашборд+вход → алерты → прод → ИИ
Параллельно: Docker/CI по мере роста
```

## 9. Риски

Разный учёт в юрлицах; нет владельца данных; «сразу ИИ» — сначала цифры.
