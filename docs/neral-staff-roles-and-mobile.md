# Сотрудники, роли и мобильные приложения

## Идея

1. **Руководство** (дашборд группы + пульт фермы) — реестр сотрудников, роли, назначение задач.
2. **Сотрудник** — отдельный вход в приложении (телефон / планшет), видит **только свои** задачи и экраны по роли.
3. **Android и Apple** — один код веб-приложения, обёртка в нативное (рекомендуемый путь ниже).

## Роли на ферме (демо)

| Код | Роль | Что видит в приложении (целевое) |
|-----|------|----------------------------------|
| `milker` | Доярка | Дойка, смена, список коров |
| `vet` | Ветеринар | Задачи, стадо, коровники, карточки |
| `driver` | Водитель | Маршруты, машины, доставка корма |
| `zootech` | Зоотехник | Кормление, DTM, сверка рациона |
| `agronomist` | Агроном | Поля, корма, силос (позже) |
| `mechanic` | Слесарь | Неисправности, техника |
| `security` | Охрана | Пропуска, смена (упрощённо) |
| `manager` | Руководитель смены | Всё + назначение задач |
| `admin` | Администратор | Сотрудники, отчёты, доступы |

## Этапы внедрения

### Этап 1 — сейчас (демо в репозитории)

- Вкладка **«Сотрудники»** на пульте Матрикс: `/matrix/staff`
- Ссылка из дашборда группы (`web/`) → управление персоналом на ферме
- Роли, фильтры, назначение задач от руководства (в `localStorage`, демо)
- Мобильная вёрстка пульта уже есть (PWA: «На экран Домой»)

### Этап 2 — вход сотрудника (демо в репозитории)

- `/matrix/login` — PIN 4 цифры + быстрый выбор сотрудника
- После входа — меню **только по роли** (доярка не видит «Машины» и т.д.)
- `/matrix/my-tasks` — задачи от руководства + для ветеринара очередь Afimilk
- Примеры PIN: ветеринар Забиров `1001`, доярка Ямалетдинов `1020`, водитель `1030`, руководитель `1070`

### Этап 3 — приложения в Store

| Вариант | Плюсы | Минусы |
|---------|--------|--------|
| **PWA** | Уже почти готово, без магазина | Нет push из коробки, слабее «как приложение» |
| **Capacitor** | Один код `web-matrix`, App Store + Google Play, камера/GPS | Нужен Apple/Google developer account |
| **React Native** | Нативный UX | Два кодовых базы, дороже |

**Рекомендация:** Capacitor поверх `web-matrix` + бэкенд (API) для входа и задач.

### Этап 4 — бэкенд

- База: сотрудники, роли, задачи, статусы
- Push-уведомления о новой задаче от руководства
- Интеграция Afimilk / DTM по ролям
