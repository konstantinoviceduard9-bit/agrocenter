## Доступ к paywall (статус)

| Шаг | Результат |
|---|---|
| Открытие модалки логина | OK (Войти → Е-mail tab) |
| Ввод email `voloffme7@gmail.com` → "Продолжить" | OK — пользователь существует (`code:33 «Такой пользователь уже существует»` на onboarding endpoint) |
| Ввод пароля → "Войти" | **FAIL** — `code:1, message: "Неправильный пароль. Попробуйте еще раз."` |
| После 3-4 попыток | Включается Google reCAPTCHA (invisible v2) с visual challenge ("Выберите все изображения, где есть автобус") |
| Дальнейшие direct API calls | `code:255, "Подтвердите, что вы не робот"` — IP/session под подозрением |

**Вывод:** Учетные данные **неверные**. Аккаунт `voloffme7@gmail.com` зарегистрирован на rusprofile, но пароль `div4223_aA` не совпадает. Это **не** captcha-блокер по существу — captcha — это лишь rate-limit реакция на серию неуспешных попыток. Реальный блокер — неправильный пароль.

**Что делать:**
1. Сбросить пароль на rusprofile (форма «Забыли пароль?» отправит ссылку на `voloffme7@gmail.com`) → проверить почту → ввести новый пароль → пересохранить credentials в vault и в `04-References/APIs.md`

3. Контактные данные (телефон, email, сайт) и **полная финансовая отчётность** под paywall — без подписки получим только summary metrics
